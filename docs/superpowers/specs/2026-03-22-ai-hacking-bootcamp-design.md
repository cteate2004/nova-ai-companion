# AI Hacking Bootcamp — Design Spec

Nova becomes a cybersecurity learning companion focused on AI/LLM/agent security, with the goal of getting the user earning money through bug bounties.

## Goals

1. Teach AI hacking from beginner to bounty-ready through a structured curriculum
2. Keep the user engaged with daily challenges delivered via push notifications
3. Surface and track real bounty programs with AI/agent scope
4. Help write quality vulnerability reports that get accepted
5. Track earnings and progress over time

## Architecture

### New Backend Module: `hacking.js`

Manages curriculum state, challenge generation, and bounty tracking. Claude generates teaching content and challenges dynamically based on the user's current module and skill level.

### New Claude Tools (6)

#### `get_curriculum`
Returns the full curriculum map with module names, status (locked/unlocked/completed), and progress percentage for each. Shows the user where they are in their learning path.

#### `complete_lesson`
Marks a lesson/sub-topic within the current module as done. When all lessons in a module are complete, marks the module as completed and unlocks the next one. Returns updated progress.

Parameters: `module_number` (int), `lesson_name` (string)

#### `get_daily_challenge`
Returns today's challenge. If one hasn't been generated yet for today, Claude generates a new one based on the user's current module and difficulty curve. Includes the challenge prompt and number of available hints.

#### `submit_challenge_answer`
User submits their answer to today's challenge. Claude evaluates it, provides feedback, assigns a score (0-100), and updates streak/points. Returns detailed feedback on what was right, what was missed, and the reference solution.

Parameters: `challenge_id` (int), `answer` (string)

#### `track_bounty`
Add or update a bounty program. Used when Nova finds a new program or the user starts working on one. Can also log earnings when a bounty pays out.

Parameters: `action` (add/update), `program_name` (string), `platform` (string), `url` (string), `status` (string), `payout_amount` (number, optional)

#### `get_hacking_dashboard`
Returns a complete overview: current module progress, streak, total points, level, challenges completed, active bounties, total earnings, and recent activity.

## Curriculum

### Module 1: LLM Basics for Hackers
How tokens, system prompts, context windows, and temperature actually work. Understanding the attack surface starts with understanding the system.

Challenge style: "What would happen if..." reasoning questions.

### Module 2: Direct Prompt Injection
Overriding system instructions, role-play exploits, encoding tricks (base64, rot13, pig latin), delimiter confusion, instruction hierarchy manipulation.

Challenge style: Given a system prompt, craft an injection that extracts it.

### Module 3: Indirect Prompt Injection
Poisoning documents, emails, and web content that agents consume. Hidden instructions in markdown, HTML comments, invisible unicode, tool output manipulation.

Challenge style: Design a payload that triggers when an agent processes external content.

### Module 4: Agent Tool Abuse
Making agents call tools with malicious parameters, parameter injection, chaining tool calls to escalate privileges, exploiting tool description trust.

Challenge style: Find the exploit path in a described agent architecture.

### Module 5: RAG Poisoning & Data Exfiltration
Extracting private documents from retrieval systems, manipulating embeddings, crafting queries that leak context, membership inference attacks.

Challenge style: Craft queries that would leak context from a RAG system.

### Module 6: Multi-Agent Exploits
Attacking agent-to-agent communication, trust boundary violations, delegation chain manipulation, confused deputy attacks in agent networks.

Challenge style: Map attack surfaces in a multi-agent system diagram.

### Module 7: Guardrail Bypasses
Understanding safety filters, content classifiers, output validators. How they work, where they fail, common bypass patterns.

Challenge style: Analyze a guardrail setup and identify weaknesses.

### Module 8: Bug Bounty Methodology
Recon for AI features, scoping agent architectures, writing quality reports, responsible disclosure process, platform-specific tips for HackerOne/Bugcrowd/Huntr.

Challenge style: Write a mock bounty report for a practice vulnerability.

## Bounty Tracking

### Discovery
Nova uses the existing Brave Search integration to search weekly for:
- New AI/LLM bounty programs on HackerOne, Bugcrowd, Huntr
- Companies that recently launched AI features (potential new scope)
- AI security news and disclosed vulnerabilities (learning material)

### Workflow
1. Nova finds program → pushes notification
2. User says they're interested → Nova adds to `hacking_bounties` as "watching"
3. User starts testing → status moves to "active"
4. Nova helps scope AI features, think through attack surfaces
5. User finds vulnerability → Nova helps draft the report
6. Submitted → tracks status until resolution
7. Payout → logs earnings in dashboard

### Report Writing
Nova helps structure findings in standard format:
- Title / Severity / CVSS
- Summary (1-2 sentences)
- Impact (what an attacker could do)
- Steps to reproduce (detailed, numbered)
- Proof of concept
- Remediation suggestion

Primary interaction is through chat — the dashboard is for at-a-glance status, but Nova teaches through conversation:

- "What's today's challenge?" → serves challenge
- "Give me a hint" → reveals next hint
- "I think the answer is..." → evaluates and gives feedback
- "Teach me about [topic]" → mentor mode
- "Help me scope [company]" → recon assistance
- "Review my report" → critiques bounty report draft
- "How am I doing?" → progress summary

### Push Notifications
Via existing scheduler and push infrastructure:

- **3:00 AM PST daily** — Daily challenge with a teaser
- **Sunday morning** — Weekly recap
- **Ad-hoc** — New bounty program alerts

## Technical Notes

- All challenge content is generated dynamically by Claude, not hardcoded. The curriculum table stores structure; Claude generates the teaching.
- Challenges are generated once per day and cached in `hacking_challenges` so the user gets the same challenge if they ask multiple times.
- The hacking tools are added to the existing Claude tool definitions in `claude.js`.
- Bounty search uses the existing `web-search.js` module.
- Push notifications use the existing `push.js` module.
- Streak tracking resets if a day is missed (no challenge completed by 11:59 PM PST).
- All teaching is strictly educational and ethical. Nova never helps attack systems without authorization.
